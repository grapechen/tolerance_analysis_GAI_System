# 本地模型 vs 雲端模型比較

> **注意**：本專案現在使用統一的 AI 助手（`ai_app.py`），可在同一介面中切換本地模型和雲端模型。

## 快速選擇指南

### 🏠 選擇本地模型如果你：
- ✅ 有強大的電腦硬體（GPU 更佳）
- ✅ 需要完全離線運作
- ✅ 對資料隱私有極高要求
- ✅ 不想產生任何費用
- ✅ 願意下載模型檔案（數 GB）

### ☁️ 選擇雲端模型如果你：
- ✅ 使用筆記型電腦或低階硬體
- ✅ 需要快速回應和高效能
- ✅ 想使用超大型模型（如 671B 參數）
- ✅ 有穩定的網路連線
- ✅ 接受合理的使用費用（有免費方案）

## 詳細比較表

| 項目 | 本地模型 | 雲端模型 |
|------|----------|----------|
| **安裝需求** | 需下載模型（數 GB） | 無需下載 |
| **首次設置** | 下載時間較長 | 登入 Ollama 帳號即可 |
| **硬體需求** | 高（建議 16GB+ RAM） | 低（任何電腦） |
| **推理速度** | 取決於硬體 | 快速（資料中心 GPU） |
| **資源佔用** | 高（CPU/GPU/記憶體） | 極低 |
| **電池消耗** | 高 | 低 |
| **網路需求** | 僅下載模型時 | 每次推理都需要 |
| **離線使用** | ✅ 完全支援 | ❌ 需要網路 |
| **費用** | 免費 | Free: $0, Pro: $20/月, Max: $100/月 |
| **隱私等級** | ⭐⭐⭐⭐⭐ 完全本地 | ⭐⭐⭐⭐ Ollama 不保留資料 |
| **模型大小限制** | 受限於本地硬體 | 可使用超大模型（最高 671B） |
| **切換方式** | 在介面右上角選擇模型 | 在介面右上角選擇模型 |

## 成本分析

### 本地模型
- **初始成本**: $0
- **運行成本**: 電費（約 $0.01-0.05/小時，取決於硬體）
- **儲存成本**: 硬碟空間（每個模型 1-10 GB）
- **總成本**: 極低

### 雲端模型
- **初始成本**: $0
- **運行成本**: 
  - Free: $0/月（有使用限制）
  - Pro: $20/月
  - Max: $100/月
- **儲存成本**: $0（無需下載）
- **總成本**: 取決於使用量

## 使用情境建議

### 情境 1：個人學習/研究
**推薦**: 本地模型
- 理由：免費、無限制使用

### 情境 2：公司內部使用
**推薦**: 本地模型
- 理由：資料隱私、離線可用

### 情境 3：外出/移動辦公
**推薦**: 雲端模型
- 理由：省電、不佔資源

### 情境 4：複雜推理任務
**推薦**: 雲端模型（超大型）
- 理由：可使用 671B 參數模型

### 情境 5：展示/Demo
**推薦**: 雲端模型
- 理由：快速啟動、穩定效能

## 混合使用策略

**推薦做法**：在同一個 AI 助手中根據需求切換模型

```bash
# 啟動 AI 助手
run_ai.bat

# 在網頁介面右上角切換模型：
# - 日常查詢 → gemma3:4b（本地，快速）
# - 複雜分析 → gpt-oss:120b-cloud（雲端，強大）
# - 離線使用 → 任何本地模型
# - 外出時 → 任何雲端模型
```

**優勢**：
- 單一介面，無需切換服務
- 靈活選擇最適合的模型
- 本地和雲端模型無縫切換

## 如何使用

### 使用本地模型

1. 安裝 Ollama 應用程式
2. 下載模型：
   ```bash
   ollama pull gemma3:4b
   ollama pull llama3.1:8b
   ```
3. 啟動 AI 助手：`run_ai.bat`
4. 在介面中選擇本地模型

### 使用雲端模型

1. 在 Ollama 應用程式中登入帳號
2. 雲端模型會自動出現（標示 ☁️）
3. 啟動 AI 助手：`run_ai.bat`
4. 在介面中選擇雲端模型

### 切換模型

在 AI 助手介面右上角的下拉選單中選擇：
- 本地模型：如 `gemma3:4b`、`llama3.1:8b`
- 雲端模型：如 `☁️ gpt-oss:120b-cloud`、`☁️ deepseek3.1:671b-cloud`

## 效能測試（參考）

| 測試項目 | 本地模型 (i7-10700K, gemma3:4b) | 雲端模型 (gpt-oss:120b) |
|----------|-------------------------------|------------------------|
| 簡單查詢回應時間 | 2-5 秒 | 1-2 秒 |
| 複雜分析回應時間 | 5-10 秒 | 2-4 秒 |
| 記憶體佔用 | 4-8 GB | <100 MB |
| CPU 使用率 | 50-80% | <5% |
| 模型載入時間 | 首次 5-10 秒 | 無需載入 |

*註：本地模型效能取決於硬體配置和模型大小*

## 總結

- **預算有限** → 本地模型
- **硬體有限** → 雲端模型
- **隱私優先** → 本地模型
- **效能優先** → 雲端模型（大型）
- **離線需求** → 本地模型
- **不確定** → 兩者都試試（在同一介面切換）

**最佳實踐**：根據任務需求在同一個 AI 助手中靈活切換本地和雲端模型！

# 1. 系統開發目標 (Objective)

解決 **「設計規格」** 與 **「製造能力」** 之間的資訊斷層：

*   **痛點**：設計師給出公差 (e.g., 0.005mm)，但不確定廠內哪台機器做得到。
*   **解方**：建立一個互動式儀表板 (`company.html`)，將機台規格數據化，實時評估適配性。
*   **核心價值**：
    1.  **降低風險**：避免指派精度不足的機台。
    2.  **提升效率**：秒級篩選，取代人工翻閱數十本型錄。

# 2. 系統介面概覽 (UI Overview)

以前端互動網頁呈現，分為三大區塊：

1.  **控制面板 (Left Panel)**
    *   **場景選擇**：模具 / 航太 / 量產 (權重動態調整)。

2.  **即時監控 (KPI Dashboard)**

3.  **推薦清單 (Result List)**
    *   依據「綜合評分」排序，提供詳細的「推論理由」。

# 3. 特別介紹：關鍵規格解析 (Key Specs Spotlight)

為了讓推薦更精準，我們特別拉出了這兩個「常被忽略但至關重要」的規格：

### A. 棒材送料能力 (Bar Capacity)
*   **定義**：車床主軸中心孔允許通過的最大棒材直徑。
*   **重要性**：決定了**「自動化量產」**的可行性。
    *   *小於此數值*：可搭配棒材送料機 (Bar Feeder)，實現 24hr 無人化連續加工。
    *   *大於此數值*：必須裁斷成單件，依賴人工或機械手臂上下料 (Chucking Work)。
*   **系統邏輯**：若選「量產模式」，此欄位權重加倍。

### B. 定位精度 (Positioning Accuracy P)
*   **定義**：機台實際移動距離與指令距離的「最大誤差值」。
*   **與重現性 (Ps) 的差異**：
    *   **定位精度**：代表**「準不準」**（決定首件尺寸是否合格）。
    *   **重現性**：代表**「穩不穩」**（決定量產時的 CPK/良率）。
*   **計算公式 (VDI 3441)**：
    *   $$ P = Pa + Ps + U $$
    *   (定位精度 P = 平均偏差 Pa + 重現性 Ps + 反向間隙 U)
    *   包含這台機器所有的系統誤差與隨機誤差之總和。
*   **系統邏輯**：系統會同時檢查這兩項。若要在不挑機台的情況下也能做對，**定位精度**必須小於設計公差。

### C. 重現性 (Repeatability Ps)
*   **定義**：同一指令多次移動到同一位置的「散佈範圍」(Dispersion)。
*   **重要性**：是**「製程能力 (Cpk)」**的基石。
    *   *定位準但不重現*：每次都在不同地方，無法量產。
    *   *重現好但定位偏*：可靠補正 (Offset) 修正。
*   **計算公式 (VDI 3441 / ISO 230-2)**：
    *   $$ P_s = 6 \cdot \sigma $$ (或是 ISO 的 $$ R = 4 \cdot s $$)
    *   也就是標準差的 4~6 倍，代表 99.7% 的落點範圍。
*   **系統邏輯**：這是最關鍵的指標。若 **重現性 > 設計公差**，代表機台本身不穩，幾乎註定會有不良品，系統會直接**淘汰**。

# 4. 核心演算法：多維度評分 (Scoring Logic)

我們不只看「能不能做」，還看「做不做的好」。

### A. 硬性篩選 (Hard Filters)
*   **一票否決制**：只要有一項規格 (如 X 軸行程不足、無此製程) 不符，直接淘汰。
*   **精度把關**：若 `機台重現性 (0.008mm) > 目標公差 (0.005mm)` -> **淘汰**。

### B. 彈性加分 (Bonus Scoring)
*   **能力比 (Capability Ratio)**：若機台能力是需求的 **2倍** 以上，給予額外加分 (代表製程餘裕度高，CPK 表現均值較佳)。
*   **規格優勢**：轉速更快、行程更大、刀庫更多，皆按比例加分。

---

# 4. 關鍵功能：智慧精度匹配 (Smart Accuracy Matching)

針對製造業最在意的「精度」進行深度處理：

1.  **資料來源標準化**：
    *   整合 **ISO 10791-4**、**VDI 3441**、**JIS B 6338** 等國際標準數據。
    *   區分「定位精度」與「重現性」不同指標。
2.  **防呆機制**：
    *   若機台**缺精度資料**，系統會標註「⚠️ 風險警告」。
    *   若應用於「模具場景」且缺資料，會執行**懲罰性扣分**。

# 5. 應用場景權重 (Scenario-based Weighting)

系統能「聽懂」使用者的任務屬性：

| 應用場景 | 權重邏輯 | 適用案例 |
| :--- | :--- | :--- |
| **通用 (General)** | 標準評分，無特殊偏好。 | 一般零件加工 |
| **模具 (Mold)** | **精度優先** (佔比極高)，**高轉速**加分。缺精度資料者直接刷掉或重罰。 | 精密模具、石墨電極 |
| **航太 (Aerospace)** | 強調 **多軸能力**、**大行程** 與 **資料信賴度**。 | 機身結構件、渦輪葉片 |
| **量產 (Mass)** | 強調 **快送速度** (Rapid Traverse)、**刀具交換效率**。 | 汽機車零件量產 |

*   **Tongtai (東台)**：VP / EZ 系列。
*   **Mazak**：FJV 系列 (標註 High Precision)。

# 7. 結論 (Conclusion)

這份 `company.html` 不僅是一個查詢網頁，更是一個 **「製造資源決策支援系統」**。

它將**老師傅的經驗**（選機邏輯、看重什麼規格）轉化為**程式碼與演算法**，確保每一次的派工決策都有數據支撐，真正落實「智慧製造」的數據驅動精神。

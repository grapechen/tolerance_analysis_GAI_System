# Ollama 雲端模型使用指南

## 什麼是 Ollama 雲端模型？

Ollama 雲端模型讓你可以在 Ollama 應用程式中直接使用雲端 GPU 運算的大型模型，無需下載模型到本地。

## 優勢

- **🚀 更快** - 使用資料中心級 GPU，推理速度更快
- **💪 更強** - 可使用超大模型（如 120B 參數的 gpt-oss）
- **💻 省資源** - 不佔用本地 CPU/GPU/記憶體
- **🔋 省電** - 筆記型電腦續航更久
- **📦 免下載** - 無需下載數 GB 的模型檔案

## 如何使用

### 步驟 1：登入 Ollama 帳號

1. 開啟 Ollama 應用程式
2. 點擊右上角設定圖示
3. 選擇 "Sign in"
4. 使用 GitHub 或 Google 帳號登入
5. 選擇方案（Free 方案即可開始使用）

### 步驟 2：在 AI 助手中使用

1. 啟動 AI 助手：`run_ai.bat`
2. 開啟瀏覽器：http://localhost:7011
3. 在右上角模型選單中，選擇帶有 ☁️ 圖示的模型
4. 開始對話！

## 可用的雲端模型

根據 Ollama 官方標記為 cloud 的模型：

### 推理模型
- **gpt-oss:20b / gpt-oss:120b-cloud** - OpenAI 開源推理模型
- **deepseek3.1:671b-cloud** - DeepSeek 超大型混合專家模型（671B 參數）

### 視覺語言模型
- **qwen3-vl** (2b, 4b, 8b, 30b, 32b, 235b) - 強大的視覺理解模型
- **qwen3-v1:235b-cloud** - 視覺語言超大模型
- **ministral-3** (3b, 8b, 14b) - 邊緣部署優化模型

### 程式碼模型
- **qwen3-coder:30b / qwen3-coder:480b-cloud** - 程式碼專用超大模型

### 通用模型
- **gemma3** (270m, 1b, 4b, 12b, 27b) - Google 高效能模型
- **glm-4.6** - 進階代理和推理模型

### 推薦使用場景
- **日常查詢** → gemma3:4b（快速）
- **複雜推理** → gpt-oss:120b-cloud 或 deepseek3.1:671b-cloud
- **程式碼相關** → qwen3-coder:480b-cloud
- **視覺理解** → qwen3-v1:235b-cloud（如需處理圖片）

**注意**：可用的雲端模型會隨時更新，請以 Ollama 應用程式中顯示的為準

## 本地模型 vs 雲端模型

| 特性 | 本地模型 | 雲端模型 |
|------|----------|----------|
| **速度** | 取決於硬體 | 快速（資料中心 GPU） |
| **模型大小** | 受限於本地硬體 | 可使用超大模型 |
| **資源佔用** | 高 | 極低 |
| **網路需求** | 不需要 | 需要 |
| **費用** | 免費 | Free 方案有限制 |
| **隱私** | 完全本地 | Ollama 不保留資料 |

## 使用限制

### Free 方案
- 每小時/每週有使用限制
- 適合輕度使用和測試

### Pro 方案（$20/月）
- 更高的使用限制
- 適合日常使用

### Max 方案（$100/月）
- 5 倍使用限制
- 適合重度使用

## 故障排除

### Q: 看不到雲端模型？
**A:** 確認已在 Ollama 應用程式中登入帳號

### Q: 雲端模型回應慢？
**A:** 可能是網路連線問題，或 Ollama 服務負載較高

### Q: 超過使用限制？
**A:** 等待限制重置，或考慮升級方案

### Q: 想要完全離線使用？
**A:** 使用本地模型，在 Ollama 應用程式中下載模型即可

## 推薦使用情境

**使用雲端模型：**
- 外出使用筆記型電腦
- 需要快速回應
- 處理複雜查詢
- 本地硬體較弱

**使用本地模型：**
- 完全離線環境
- 對隱私有極高要求
- 有強大的本地硬體
- 不想產生費用

## 混合使用策略

你可以根據情境靈活切換：

1. **日常查詢** → 本地小模型（如 gemma3:4b）
2. **複雜分析** → 雲端大模型（如 gpt-oss:120b）
3. **外出時** → 雲端模型（省電）
4. **離線時** → 本地模型
